# 🧠 LeetCode Playbook

A structured approach to solving LeetCode problems by focusing on patterns, reasoning, and iterative improvement—not memorization.

This repository documents my journey of building strong algorithmic intuition by recognizing recurring problem structures and refining solutions over time.

---

## 🗂️ Structure

Each problem is organized to separate thinking stages: <br>
problems/ <br>
problem_name/ <br>
solution.py     → final optimized solution <br>
notes.md        → reasoning, mistakes, and insights <br>
attempts.ipynb  → experimentation space

patterns/
Notes on reusable algorithmic strategies, such as:

* Sliding Window
* Binary Search
* Dynamic Programming

progress_log.md
Weekly reflection on learning progress and challenges

---

## 🧠 Learning Approach

Instead of focusing on quantity, I focus on:
- Understanding why a solution works
- Identifying reusable patterns across problems
- Reducing brute-force thinking over time
- Turning mistakes into structured insights

---

## 🔍 Core Patterns

This repository is built around mastering:
- Sliding Window → efficient range tracking
- Binary Search → structured elimination of search space
- Dynamic Programming → reusing overlapping subproblems

---

## 🚀 Goal

To develop a strong, transferable problem-solving intuition that goes beyond individual LeetCode problems and builds real algorithmic thinking.

---

## 📌 Note

This is a living repository. Solutions and notes evolve as understanding deepens.
